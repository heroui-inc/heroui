## Latest Release


### v3.0.0-beta.1

**November 5, 2025**

#### Major Redesign
- Complete redesign of HeroUI v3 design system, merging the soul of v2 with v3's simplicity
- New [Figma Kit V3](https://www.figma.com/community/file/1546526812159103429/heroui-figma-kit-v3) available
- All components redesigned with improved animations and accessibility

#### 8 New Components
- **Alert**: Status messages and notifications
- **Checkbox & CheckboxGroup**: Form checkbox inputs
- **InputOTP**: One-time password input
- **Listbox**: List of selectable options
- **Select**: Dropdown selection component
- **Slider**: Range input component
- **Surface**: Base surface container component

#### Design System Updates
- **(‚ö†Ô∏è Breaking change)**: `--panel` replaced with `--surface` and `--overlay`
- **(‚ö†Ô∏è Breaking change)**: Removed `--surface-1`, `--surface-2`, `--surface-3` variables
- **(‚ö†Ô∏è Breaking change)**: Default border width changed from `1px` to `0px`
- **(‚ö†Ô∏è Breaking change)**: Shadow system redesigned with `--shadow-surface` and `--shadow-overlay`
- New background shade levels and soft color variants
- Refined status colors (success, warning, danger)

#### Component API Changes
- **(‚ö†Ô∏è Breaking change)**: Chip `type` prop renamed to `color`, added `size` prop
- Link component enhanced with `underline` and `underlineOffset` props
- Improved TypeScript types across all components

#### Flexible Component Patterns
- Dual syntax support: use components with or without `.Root` suffix (e.g., `<Avatar>` or `<Avatar.Root>`)
- Three equivalent patterns: compound without `.Root`, compound with `.Root`, named exports
- Backward compatible with alpha.35 `.Root` pattern
- **(‚ö†Ô∏è Breaking change)**: Type references require object syntax (`Avatar["RootProps"]` instead of `Avatar.RootProps`)
- **(‚ö†Ô∏è Breaking change)**: Renamed `TabListWrapper` ‚Üí `TabListContainer` across all exports

#### Performance Improvements
- Tailwind CSS v4 migration for better performance
- CSS native animations for smoother transitions
- Reduced bundle size and faster runtime

## Previous Releases

### v3.0.0-alpha.35

**October 21, 2025**

#### React Server Components Support
- Fixed critical issue where compound components didn't work in React Server Components (RSC)
- Moved compound pattern logic from components to index files, resolving `"use client"` conflicts
- **(‚ö†Ô∏è Breaking change)**: Main component now requires `.Root` suffix (e.g., `<Avatar>` ‚Üí `<Avatar.Root>`)
- Named exports remain unchanged and fully supported

#### React 19 Improvements
- Removed `forwardRef` (now native in [React 19](https://react.dev/blog/2024/12/05/react-19#ref-as-a-prop))
- Simplified Context usage (`Context.Provider` ‚Üí [React 19](https://react.dev/blog/2024/12/05/react-19#context-as-a-provider))

#### Switch Component Refactoring
- **(‚ö†Ô∏è Breaking change)**: Split Switch and SwitchGroup into separate components
- Cleaner API: `<SwitchGroup>` replaces `<Switch.Group>` and `<Switch.GroupItems>`
- Matches Radio/RadioGroup pattern for consistency
- Separate styles, types, and implementations

#### Affected Components
All compound components now require `.Root` suffix: `Accordion`, `Avatar`, `Card`, `Disclosure`, `Fieldset`, `Kbd`, `Link`, `Popover`, `Radio`, `Switch`, `Tabs`, `Tooltip`
