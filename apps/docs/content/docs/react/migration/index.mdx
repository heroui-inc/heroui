## Overview

### Migration MCP Tool

To make migration easier, we provide a **Migration MCP (Model Context Protocol) server** that AI assistants can use to access migration guides directly. This tool provides programmatic access to all migration documentation, making it easier for AI-powered development tools to help you migrate your codebase.

**Features:**
- Access to comprehensive migration guide
- Component-specific migration guides
- Step-by-step migration instructions
- Code examples and API changes

**Setup:**

Add the Migration MCP server to your MCP client configuration (e.g., in Cursor or other MCP-compatible tools):

```json
{
  "mcpServers": {
    "heroui-migration": {
      "url": "https://migration-mcp.heroui.com"
    }
  }
}
```

<Callout type="warning">
  **Important:** Do not use the Migration MCP alongside the HeroUI React MCP server. Having both configured simultaneously may confuse AI assistants about which set of component documentation to use. For migration purposes, connect only to the Migration MCP. Once migration is complete, you can switch to using the HeroUI React MCP for v3 component documentation.
</Callout>

Once configured, AI assistants can automatically access migration guides through tools to help you migrate your components more efficiently. The MCP server also provides guided migration prompts that can be used with slash commands in your agent chat.

**Repository:** The Migration MCP server code is available at [`heroui-inc/heroui-mcp`](https://github.com/heroui-inc/heroui-mcp/tree/main/apps/migration-mcp).

## Choose Your Migration Approach

HeroUI v2 and v3 cannot coexist in the same project without special setup. You have two migration approaches to choose from:

### Phased Migration

**Best for:** Projects that can dedicate focused time to complete migration, teams comfortable working in feature branches with temporarily broken code.

**How it works:**
- Migrate all component code first (project will be broken during this phase)
- Switch dependencies to v3
- Complete styling migration

**Advantages:**
- Simpler setup - no complex coexistence configuration
- Cleaner transition - only one version active at a time
- No CSS conflicts during migration
- Supported by Migration MCP prompts

**Disadvantages:**
- Project is broken during migration
- Requires working in a feature branch
- All components must be migrated before switching dependencies

**Get started:** [Phased Migration Guide](/docs/react/migration/phased-migration)

### Incremental Migration with Coexistence

**Best for:** Projects that need to stay functional during migration, teams migrating gradually over time, large codebases migrating feature-by-feature.

**How it works:**
- Set up coexistence using pnpm aliases or component packages
- Migrate components one-by-one while keeping project functional
- Complete migration and remove v2 dependencies

**Advantages:**
- Project remains functional during migration
- Can migrate gradually over time
- Can test v3 components alongside v2

**Disadvantages:**
- More complex initial setup
- Both CSS systems loaded (larger bundle size)
- Potential styling conflicts
- Requires managing two versions simultaneously

**Get started:** [Incremental Migration Guide](/docs/react/migration/incremental-migration)

### Quick Comparison

| Aspect | Phased Migration | Incremental Migration |
|--------|-----------------|----------------------|
| Project State | Broken during migration | Functional throughout |
| Setup Complexity | Simple | Complex (coexistence setup) |
| CSS Conflicts | None | Possible |
| Bundle Size | Normal | Increased during migration |
| Migration Speed | Faster (focused) | Slower (gradual) |
| Best For | Focused migration sessions | Gradual migration over time |

## High-Level Migration Checklist

Regardless of which approach you choose, you'll need to:

### Dependencies
- Update React to v19+
- Update HeroUI packages to v3
- Remove Framer Motion
- Update Tailwind CSS to v4

### Configuration
- Remove `heroui()` plugin from Tailwind config
- Update CSS imports (`@import "tailwindcss"; @import "@heroui/styles";`)
- Remove `hero.ts` file (if exists)

### Application Code
- Remove `HeroUIProvider` wrapper
- Update component imports and APIs
- Migrate hooks (see [Hooks Migration Guide](/docs/react/migration/hooks))
- Update styling (see [Styling Migration Guide](/docs/react/migration/styling))

### Testing
- Visual regression testing
- Functionality testing
- Accessibility testing
- Performance testing

For detailed step-by-step instructions, see the [Phased Migration Guide](/docs/react/migration/phased-migration) or [Incremental Migration Guide](/docs/react/migration/incremental-migration).

<section id="major-changes">

## Major Changes

- **No Provider Required**: v3 removes the need for `HeroUIProvider`
- **CSS-First Theming**: New CSS-based theming system replaces Tailwind plugin
- **React 19 Required**: v3 requires React 19+ (v2 supports React 18+)
- **No Framer Motion**: Animation library dependency removed
- **Component API Updates**: Many components use React Aria Components patterns
- **Compound Components**: New compound component patterns for better customization
- **Hooks Removed**: v2 component hooks like `useSwitch` or `useInput` are removed - use compound components instead. `useDisclosure` is replaced with `useOverlayState`. See the [Hooks Migration Guide](/docs/react/migration/hooks) for details.

</section>

Many components now use compound component patterns for better composition and flexibility. For example:

- **Checkbox**: `Checkbox`, `Checkbox.Control`, `Checkbox.Indicator`, `Checkbox.Content`
- **Card**: `Card`, `Card.Header`, `Card.Title`, `Card.Description`, `Card.Content`, `Card.Footer`

- **Select**: `Select`, `Select.Trigger`, `Select.Value`, `Select.Content`, `Select.Item`
- **Dropdown**: `Dropdown`, `Dropdown.Trigger`, `Dropdown.Content`, `Dropdown.Item`

See specific component migration guides for detailed examples and usage patterns.

## Common Issues and Solutions

### Issue: Components not rendering

**Solution:** Ensure you've imported `@heroui/styles` in your CSS file and it's imported after `tailwindcss`.

### Issue: Styles not applying

**Solution:** Check that Tailwind CSS v4 is properly configured and CSS imports are in the correct order.

### Issue: TypeScript errors

**Solution:** Update your TypeScript types. v3 may have different prop types. Check component documentation.

### Issue: Animations not working

**Solution:** v3 handles animations differently. Check if the component supports animations in v3.
